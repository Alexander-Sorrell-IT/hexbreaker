Real measurement of Hexbreaker Court on the **NIST CFReDS Hacking Case** disk
image (`4Dell_Latitude_CPi.E01`). This directory is the committed, verifiable
evidence behind every "NIST 4/4" claim in the docs. Generated by
[`scripts/court_on_nist_fsm.py`](../../scripts/court_on_nist_fsm.py); reproducible
from the real disk with the commands below.

## What this measures (and what it does NOT)

- **Real disk, real tools.** Evidence is the genuine RECYCLER `fls` listing + the
  real `INFO2` index extracted off the `.E01` with `icat` — not synthesized. Each
  CONFIRMED finding is corroborated by **two independent tool steps** (`fls` shows
  the `Dc<N>` slot exists; `INFO2` maps that slot to the original path).
- **Recovery, not blind discovery.** The agent reads the INFO2 index (which lists
  all four original paths) and corroborates each entry. This is what a forensic
  examiner does — it is *confirmation/recovery of deleted artifacts named in real
  evidence*, **not** discovery of unknown evil from nothing.
- **`max_rounds = 4` is derived from real evidence, not the answer key.** The loop
  runs once per deleted-exe slot visible in the INFO2 record
  (`_info2_original_exe_paths()` reads the same INFO2 bytes shown to the agent). A
  neutral examiner reading INFO2 sees the same 4 slots.
- **The answer key never reaches the agent.** `load_case` reads `answer_key.json`
  and the runner immediately **discards** it (`manifest, _ = load_case(...)`); its
  scorer-only content never enters any Prosecutor or Defender prompt. `answer_key.json`
  is consumed **only by the scorer, after the run**. Both this property and the
  `max_rounds`-from-INFO2 property are enforced by
  `tests/test_court_on_nist_fsm_honesty.py`.
- **Precision 1.0 here is recovery accuracy, NOT bait-resistance.** This NIST case
  has **no planted decoys** (`answer_key.json` → `decoys: []`, `planted: []`), so
  `precision = 1.0` / `fp_planted = 0` mean the Court recovered the right artifacts —
  not that it resisted baited evidence. Bait-resistance is measured separately on the
  Forge provocateur sweeps (`fp_planted = 0/80`). Do not read NIST precision as
  adversarial robustness.
- **Scope: 1 of ~31 NIST question families.** This is the recycle-bin question
  (NIST Q28) only. It is **not** a 31-question NIST F1.
- **`artifact_kind = "other"`.** The recycle bin proves **deletion**, not
  **execution**. We do not upgrade these to "executed".
- **Single-finding mode caps at 1/4.** With `max_rounds = 1` the Court confirms one
  slot (typically the first, `lalsetup250.exe`) and stops — recall 1/4. The
  multi-round loop ("already accused — name a DIFFERENT artifact") is what lifts
  recall to 4/4. This contrast is the honest motivation for the loop.

> **Not the withdrawn number.** This is unrelated to the withdrawn
> `scripts/court_on_nist.py` 95.08% batched figure, which injected ground-truth
> answers into the prompt. This path has a Defender, an FSM, a deterministic Judge,
> a hash chain, and an HMAC signature — and reads zero answers.

## Reproduce from the real disk

```bash
# 1. Mount the E01 and stage the real extracts
mkdir -p /tmp/nist-ewf-mount /tmp/nist-extracts
ewfmount datasets/nist-hacking-case/4Dell_Latitude_CPi.E01 /tmp/nist-ewf-mount
mmls /tmp/nist-ewf-mount/ewf1                       # confirm offset 63
fls -o 63 -r -p /tmp/nist-ewf-mount/ewf1 | grep -i RECYCLER > /tmp/nist-extracts/fls_recycler.txt
icat -o 63 /tmp/nist-ewf-mount/ewf1 11850 > /tmp/nist-extracts/INFO2

# 2. Run the multi-round Court (max_rounds = INFO2-visible exe slots = 4)
export DEEPSEEK_API_KEY=sk-...
PYTHONPATH=src python scripts/court_on_nist_fsm.py --case-dir /tmp/nist_court/case

# 3. Verify any committed run's chain + HMAC signature
export HEXBREAKER_HMAC_PASSWORD=hexbreaker-nist-fsm
hexbreaker verify --transcript samples/nist_fsm_run/run1/transcript.jsonl --hmac
```

**HMAC password for the committed `.sig` files:** `hexbreaker-nist-fsm`.

## Files

- `case/` — the shared case the runs were built from (`manifest.json`,
  `answer_key.json` (scorer-only), `mock_outputs/{fls.txt,info2.txt}` = the real
  extracts shown to the agent).
- `run{1..5}/` — per run: `transcript.jsonl` (+ `.sig`), `findings.json`,
  `score.json`, `transcript.outputs/` (per-tool stdout/stderr sidecars; their bytes
  are hash-checked by `verify`).
- `single_finding_baseline/` — one `max_rounds=1` run on the same evidence/code,
  confirming a single slot (`lalsetup250.exe`, recall 0.25). This is the baseline
  the multi-round loop lifts from (0.25 → 1.0), committed so the lift is reproducible.
